python虚拟环境

在Python开发中，虚拟环境（Virtual Environment）是一个非常重要的概念，它允许你为每个项目创建独立的环境，避免不同项目间的依赖冲突。以下是一些常用的虚拟环境相关命令：

  1. 安装 virtualenv：

    • pip install virtualenv：安装virtualenv工具，这是最常用的Python虚拟环境管理器之一。

  2. 创建虚拟环境：

    • virtualenv <环境名称>：在当前目录下创建一个新的虚拟环境。例如，virtualenv myenv会创建一个名为myenv的虚拟环境。

  3. 激活虚拟环境：

    • macOS/Linux：source <环境名称>/bin/activate。例如，source myenv/bin/activate。
    • Windows：<环境名称>\Scripts\activate。例如，myenv\Scripts\activate。

  4. 退出虚拟环境：

    • deactivate：无论在哪个操作系统上，退出虚拟环境的命令都是deactivate。

  5. 安装包到虚拟环境中：

    • 在虚拟环境激活的情况下使用pip install <包名>命令安装所需的包。

  6. 列出虚拟环境中安装的包：

    • pip list：在虚拟环境激活的状态下执行，列出所有已安装的包。

  7. 创建虚拟环境的需求文件：

    • pip freeze > requirements.txt：这会创建一个requirements.txt文件，其中包含了虚拟环境中所有安装包的精确版本，便于在另一环境中重建虚拟环境。

  8. 使用requirements.txt安装依赖：

    • pip install -r requirements.txt：在虚拟环境激活的状态下执行，可以根据requirements.txt文件安装所有指定版本的包。

  9. 删除虚拟环境：

    • 虚拟环境只是目录而已，所以删除虚拟环境只需删除该目录即可。例如，rm -rf <环境名称>（在macOS/Linux上）或rmdir /s /q <环境名称>（在Windows上）。

  10. 使用Python内置的 venv：

    • Python 3.3及以上版本内置了venv模块，可以直接使用而无需安装virtualenv。创建和激活虚拟环境的命令与virtualenv略有不同：
      • 创建虚拟环境：python3 -m venv <环境名称>
      • 激活虚拟环境（macOS/Linux）：source <环境名称>/bin/activate
      • 激活虚拟环境（Windows）：<环境名称>\Scripts\activate
